# TimeTrackApp — 勤怠管理システム

ブラウザだけで動く、サーバー不要の勤怠管理ツールです。労働者の打刻（出退勤・休憩）、
勤怠表の提出・承認、出勤予定の申請、雇用契約の管理、お知らせ配信までを1セットにまとめています。
データは各ブラウザの `localStorage` に保存されます（外部サーバーへの送信はありません）。

## 画面構成

| ファイル | 役割 |
|---|---|
| `kintai_user.html` | 労働者用（打刻・記録編集・出勤予定入力/提出・勤怠表提出・お知らせ閲覧） |
| `kintai_admin.html` | 管理者用（労働者単位の確認・承認/差し戻し・給与支給・お知らせ配信・設定） |
| `kintai_core.js` | 両画面共通のロジック（データ層・集計・認証・PDF・通知など） |
| `kintai.css` | 両画面共通のスタイル |
| `.htaccess` / `.htpasswd` | Apache系サーバー用のBasic認証（任意） |

## 主な機能

### 労働者（ユーザー）画面
- ワンタップ打刻（出勤 / 退勤 / 休憩 / 再開）と当日サマリー
- 「＋休暇を登録」の隣にクイックボタン：**勤怠カレンダー / 勤怠表提出 / 雇用契約**
- 今月の記録の**編集・追加・削除**
- **出勤予定カレンダー**の入力と提出（管理者へ申請）
- 勤怠表の提出（**提出前に確認ポップアップ**）
- **入力漏れ通知**：出勤予定日に打刻・休暇が無いとポップアップで通知（メール報告＝メーラーを開く方式）
- **お知らせ**表示と、固定の提出日を過ぎた際の**提出リマインド**

### 管理者画面
- **労働者単位**でのデータ確認（上部のプルダウンで切替、労働者の追加・削除）
- **通知センター**：勤怠表の提出（確認待ち）、承認済みの給与支給予定を一覧表示
- 勤怠表の**承認 / 差し戻し（コメント付きモーダル）/ 給与支給済**
- 出勤予定の申請を**確定して稼働日へ反映**
- 月次集計・法令ベースのエラーチェック・稼働日カレンダー入力
- **お知らせ配信**（重要ピン留め対応）
- **設定**：勤怠表の提出日 / 給与支給日 / 管理者メール / 会社名

## 集計・法令メモ
- 実働・時間外（1日8時間超）・深夜（22:00〜翌5:00）を自動集計
- 休憩不足（実働6h超45分・8h超60分）・退勤打刻漏れ・長時間労働・月45時間超の時間外を警告
- 勤怠記録は労働基準法により原則5年（当分の間3年）の保存義務があります
- 割増賃金の計算・36協定・就業規則の適用は社会保険労務士にご確認ください（本ツールは記録・集計の補助です）

## 使い方（ローカル）
静的ファイルのため、任意のWebサーバーに置くだけで動作します。ローカル確認例：

```bash
python -m http.server 8000
# http://localhost:8000/kintai_user.html を開く
```

`file://` で直接開くこともできますが、Basic認証や一部ブラウザの制約を避けるためWebサーバー経由を推奨します。

## データについて（重要）
- データはブラウザの `localStorage` に保存されます。**ブラウザ／端末が変わるとデータは共有されません。**
- 複数の労働者を扱う場合は、同一ブラウザ上で管理者が労働者を登録・切替して運用します。
- バックアップは管理画面の **CSV / 勤怠表PDF** を利用してください。
- メール通知は端末のメールソフトを開く方式（`mailto:`）です。自動送信にはサーバー／送信サービスが別途必要です。

## セキュリティ
- ログイン画面はクライアント側の簡易ゲートであり、本格的なアクセス制御ではありません。
- 公開サーバーに置く場合は `.htaccess` + `.htpasswd`（Basic認証）などサーバー側の保護を併用してください。
- `.htpasswd` は流出防止のため `.gitignore` に含めています（リポジトリにはコミットされません）。
